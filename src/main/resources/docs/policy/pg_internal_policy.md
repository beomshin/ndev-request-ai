# [PG 내부 정책] 신규 지불수단 연동 및 R&R 판별 가이드

## 2.1 사전 필수 정책 체크리스트 (22개 항목)
신규 지불수단을 연동하기 위해서는 아래 22가지 정책에 대한 정의가 기획서(요구사항 정의서)에 명시적으로 정의되어야 합니다. AI는 현업이 요청서를 최종 제출하기 전, 대화를 통해 누락된 항목을 역추적하여 보완해야 합니다.

1. **서비스 제공 채널:** 온/오프라인 채널 구분 (Web/App, QR/바코드 등)
2. **연동 지불수단:** 신용카드 기반 / 자체 머니 / 포인트 / 캐시 등 수단 정의
3. **연동 결제 방식:** 일반 단건 결제 / 빌링키 발급을 통한 정기결제(자동결제) 여부
4. **서비스 제공 채널/방식:** PG 통합 결제창 내 노출 / 가맹점 자체 UI를 통한 다이렉트(Direct) 호출 지원 여부
5. **인증 여부:** 본인인증(비밀번호, 생체인증 등) 필수 여부 / 비인증 결제 지원 여부
6. **인증 주체:** 인증 프로세스를 처리하고 검증하는 주체 (자사 시스템 / 제휴 원천사)
7. **승인 주체:** 최종 결제 승인 API를 호출하고 처리하는 주체 (자사 시스템 / 제휴 원천사)
8. **매입 주체:** 카드사 매입 요청을 수행하는 주체 (자사 또는 원천 제휴사)
9. **연동 VAN:** 승인/매입 처리를 위한 연동 VAN사 지정 (예: NICEVAN, KISVAN 등)
10. **할부결제 지원 여부:** 신용카드 연동 시 할부 기능 제공 여부 및 최대 할부 개월 수 제한 정의
11. **복합결제 지원 여부:** 제휴사 머니 + 캐시, 머니 + 신용카드 등 하이브리드 결제 조합 허용 여부
12. **카드사 포인트 결제 지원 여부:** 세이브 포인트 등 카드사 포인트 사용 허용 여부
13. **최소 결제 금액:** 해당 지불수단으로 결제 가능한 최소 하한선 금액 정의 (예: 1,000원 제한)
14. **상점 분담 무이자 지원 여부:** 프로모션 진행 시 상점이 수수료를 분담하는 무이자 할부 적용 여부
15. **영중소 지원 여부:** 영세/중소 우대 수수료율 적용 대상 지불수단인지 확인
16. **부분취소 지원 여부:** 결제 완료 후 금액의 일부만 취소 가능한지 여부 (★과거 연동 시 가장 자주 누락되어 민원을 유발하는 예외 포인트. 불가능 시 전체 취소 후 재결제 정책 수립 필수)
17. **취소 기한:** 결제 후 취소가 가능한 최종 데드라인 정의 (예: D+90일, 당일 취소만 가능 등)
18. **복합결제 취소 정책:** 복합결제 거래 취소 시 환불 금액의 우선순위 (ex: 포인트 선환불 후 카드 취소 등)
19. **제휴사 망취소 제공 여부:** 타임아웃 발생 시 제휴사 시스템에서 망취소(Cancel) API를 제공하는지 여부
20. **복합과세 지원 여부:** 공급가액, 봉사료, 부가세, 면세액 등 복합 과세 분리 처리가 필요한 지불수단인지 정의
21. **거래 대사 방식:** 일일 정산을 위한 대사 파일(정산 대조 데이터) 제공 방식 및 파일 데이터 규격
22. **현금영수증 발행 주체:** 머니/계좌이체 등 현금성 지불수단 결제 시 현금영수증 발행 주체 (PG사 대행 / 원천 제휴사 직접 발행 / 해당 없음)

## 2.2 PG개발실 R&R 선긋기 (자동 반려 규칙)
현업과의 인터뷰 대화 중 아래 케이스가 발견되면 AI는 요구사항 정의서 문서를 생성하지 않고 **"PG개발실에서 개발하는 부분이 아닙니다"**라는 안내와 함께 즉시 대화를 종료(반려)해야 합니다.

- **디자인/단순 UI 퍼블리싱 영역:** "결제창 버튼 색상을 빨간색에서 파란색으로 바꾸고 싶어요" (단순 CSS/레이아웃 변경건) ➡️ **[디자인팀/퍼블리싱 파트]**로 문의하도록 가이드.
- **원천사/제휴사 자체 시스템 영역:** "페이코나 카카오페이 앱 안에서 우리 서비스 쿠폰 다운받는 화면 만들어주세요" (외부 플랫폼 자체 개발건) ➡️ **[제휴 마케팅팀]**을 통해 원천사에 요청하도록 가이드.
- **단순 설정/운영 업무 영역:** "특정 가맹점 ID(MID) 비밀번호 초기화 해주세요" 또는 "정산 데이터 엑셀로 뽑아주세요" ➡️ **[결제 운영/정산 파트]**로 이관 안내.

## 2.3 [History] 과거 개발건 주요 누락 이슈 리포트
- **사례 A (카카오페이 연동 건):** 현업에서 '부분취소 불가'라는 제휴사 수수료 정책을 사전 인지하지 못하고 통합 결제창에 무조건 연동했다가, 오픈 후 고객 환불 민원 발생 시 전체 취소 후 재결제를 유도해야 하는 운영 공수가 크게 발생함. *(사전 방지 대책: 16번 항목 필히 초기 확인 요망)*
- **사례 B (네이버페이 다이렉트 건):** 현업은 다이렉트 호출(4번)을 원했으나, 보안 인증 주체(6번) 스펙 협의가 누락되어 개발 도중 API 구조를 전면 수정, 일정이 2주 지연됨.
