- **추진 배경:** 모바일 카드 결제창에서 인증 후 승인 단계 전환 지연으로 결제 포기가 다수 발생. 가맹점 CS로 "결제가 안 된다" 문의가 주간 평균 40건 접수. 결제창 타임아웃·로딩 처리 개선 요청.
- **추가 정보:** 주로 안드로이드 웹뷰 환경에서 발생 추정. / **[자동] 연동 정책 매핑 결과:** P-304(대상 채널 웹/모바일), P-408(승인 타임아웃 30s), P-404(최소 결제 금액) 검토 대상

## 3. 요청 사항
- **목표일정:** 2026-06-30
- **목표일정 근거:** 6월 말 여름 시즌 프로모션 트래픽 급증 전 안정화 필요.
- **제품명:** PG표준결제창(카드)
- **문제점 및 개선점:** [문제] 인증 완료 후 승인 응답까지 평균 12초 지연, 사용자 결제창 이탈률 약 18%. [개선] 승인 대기 로딩 UX 개선 + 타임아웃 정책 점검으로 이탈률 8% 이하 목표.
- **문제를 확인하는 방법:** 1) 안드로이드 웹뷰에서 카드 결제 진행(`goPay()` 결제창 호출) → 2) 인증 응답(`AuthResultCode=0000`, `AuthToken`/`NextAppURL` 수신) 후 `NextAppURL` 승인 POST 응답이 지연(평균 12초) → 3) 결제창 자동 종료/사용자 이탈 로그 확인. 지연 시 `NetCancelURL` 망취소 트리거 여부도 함께 점검.
- **문제 발생 빈도 (정량적):** 일 평균 카드 결제 3,500건 중 약 630건(18%) 승인 전 이탈, CS 문의 주 40건.
- **경쟁사 정보:** 주요 PG사 결제창 승인 응답 평균 3~5초 수준으로 안내.

## 4. 기대 효과
- **예상수익:** 이탈률 18%→8% 개선 시 일 약 350건 결제 추가 성사, 객단가 4.5만원 기준 일 약 1,575만원 거래 회복.
- **예상손해:** 미개선 시 프로모션 기간 이탈 누적으로 가맹점 거래액 약 10% 감소 우려.
- **기대효과:** 결제창 이탈률 절반 감소, 가맹점 CS 문의 주 40건→15건 이하 절감.
